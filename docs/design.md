# シナチェン！ 設計ドキュメント

## 利用技術 (Technology Stack)

本アプリケーションは、ブラウザ上で完結するシングルページアプリケーション(SPA)として構築されています。

*   **フロントエンド・フレームワーク**: React 19
*   **ビルドツール**: Vite 6
*   **言語**: JavaScript (ES Modules, JSX)
*   **PDF解析**: `pdfjs-dist` (v6系)
*   **差分比較**: `diff` (v9系)
*   **スタイリング**: プレーンCSS (`index.css`)

## 概要設計 (High-Level Design)

*   **フロントエンド**
    *   ユーザーからのPDFファイルアップロードを受け付け、ブラウザ内でテキストの抽出および差分の計算を行います。
    *   状態管理やUIの描画はReactを用いて行い、差分結果を視覚的に分かりやすくハイライト表示します。
*   **バックエンド**
    *   本アプリケーションにはバックエンド（サーバーサイド処理）は存在しません。ファイルのパースから差分比較まで、すべての処理はクライアント（ブラウザ）上でローカルに実行されるアーキテクチャとなっています。

## 詳細設計 (Detailed Design)

### フロントエンドのパッケージ・コンポーネント設計

#### 1. コンポーネント (Components)
*   **`src/App.jsx`**
    *   **役割**: アプリケーションのメインコンポーネント。状態管理（選択されたファイル、比較結果、ローディング状態、エラーメッセージ）を担います。
    *   **UI構成**:
        *   **アップロード領域**: 新旧のPDFファイルをドラッグ＆ドロップまたはクリックで選択できる領域。
        *   **コントロール領域**: 比較実行ボタンや、差分表示の凡例（追加、削除、TRPG的重要変更）を表示します。
        *   **差分表示領域**: 比較結果をDOMとしてレンダリングし、ページ番号（PAGE_MARKER）やテキストの変更箇所を色分けして表示します。

#### 2. ユーティリティモジュール (Utils)
*   **`src/utils/pdfParser.js`**
    *   **役割**: `pdfjs-dist` を利用して、アップロードされたPDFファイルからテキストデータを抽出します。
    *   **処理詳細**: PDFの各ページをループ処理し、ページ内のテキストをY座標などを考慮しつつ結合します。ページ間に独自のマーカー（`[---PAGE_MARKER: {ページ番号}---]`）を挿入し、後続の差分表示でページ区切りを認識できるようにします。Vite環境に合わせたPDFワーカーの読み込み設定もここで行います。
*   **`src/utils/diffEngine.js`**
    *   **役割**: `diff` ライブラリを利用して、抽出された2つのテキストデータ間の行単位の差分（diff）を計算します。
    *   **処理詳細**: `diffLines` によって計算された追加（`added`）・削除（`removed`）の配列を整形します。同時に、変更されたテキストにTRPG固有の重要キーワード（例：「判定」「ダイス」「SAN」「技能」など）が含まれているかを判定し、含まれる場合は `isImportant` フラグを付与して返却します。
